### Run Tests
python manage.py test lidar_app.tests -v 2

## FILE UPLOAD & AUTO-PROCESSING

When a user uploads a LAS/LAZ file:
1. File is saved to MEDIA_ROOT/lidar_uploads/
2. LidarDataset record is created with status='uploaded'
3. Django signal triggers automatic processing
4. LidarProcessor.extract_features() extracts all metrics
5. ForestCompartment records created with metrics
6. Status updated to 'completed' or 'error'

No manual processing needed - automatic on upload!

## API ENDPOINTS

### DATASETS ENDPOINTS

1. List All Datasets
   GET /api/lidar/datasets/
   Query params:
   - search: Filter by name/description
   - ordering: -upload_date, name, processing_status
   Response: Paginated list of datasets

2. Upload New Dataset
   POST /api/lidar/datasets/upload/
   Content-Type: multipart/form-data
   Body:
   {
       "file": <LAS or LAZ file>,
       "name": "Dataset Name",
       "description": "Optional description"
   }
   Response: Created dataset object

3. Get Dataset Details
   GET /api/lidar/datasets/{id}/
   Response: Full dataset with all metrics

4. Get Processing Status
   GET /api/lidar/datasets/{id}/status/
   Response: 
   {
       "id": 1,
       "name": "Dataset Name",
       "processing_status": "completed",
       "metrics": {
           "point_count": 1000000,
           "min_height": 0.0,
           "max_height": 45.3,
           "avg_point_density": 10.5,
           "extent": "POLYGON(...)"
       }
   }

5. Get All Metrics
   GET /api/lidar/datasets/{id}/metrics/
   Response:
   {
       "dataset": {...},
       "aggregated_metrics": {
           "total_compartments": 100,
           "total_area_hectares": 1000,
           "avg_canopy_height": 24.5,
           "max_canopy_height": 45.3,
           "avg_canopy_cover": 87.3,
           "avg_foliage_diversity": 2.1
       }
   }

6. Get Point Cloud (3D Viewer)
   GET /api/lidar/datasets/{id}/point-cloud/
   Returns: 100k downsampled points with x, y, z coordinates
   Response:
   {
       "dataset_id": 1,
       "dataset_name": "Dataset",
       "point_count": 100000,
       "points": {
           "x": [x1, x2, ...],
           "y": [y1, y2, ...],
           "z": [z1, z2, ...],
           "intensity": [int1, int2, ...],
           "classification": [class1, class2, ...]
       }
   }

7. Get Compartments
   GET /api/lidar/datasets/{id}/compartments/
   Response:
   {
       "dataset_id": 1,
       "dataset_name": "Dataset",
       "compartments": [
           {
               "id": 1,
               "name": "Compartment_0_0",
               "area_hectares": 10.5,
               "geometry": {...},
               "metrics": {...}
           }
       ],
       "total_count": 100
   }

8. Manually Process Dataset
   POST /api/lidar/datasets/{id}/process/
   (Usually auto-processed, but can trigger manually)

9. Delete Dataset
   DELETE /api/lidar/datasets/{id}/

## METRICS EXTRACTED

### Dataset-Level Metrics
- point_count: Total number of points in dataset
- min_height: Minimum point height (m)
- max_height: Maximum point height (m)
- avg_point_density: Average points per m² 
- extent: Spatial bounding box POLYGON (WGS84)

### Compartment-Level Metrics
- canopy_height_mean: Average height of all points (m)
- canopy_height_max: Maximum height in compartment (m)
- height_percentile_95: 95th percentile height (m)
- height_percentile_75: 75th percentile height (m)
- height_percentile_50: Median height (m)
- canopy_cover_percent: % of points > 2m threshold
- crown_volume: Estimated volume of tree crowns (m³)
- foliage_height_diversity: Std deviation of canopy heights
- area_hectares: Compartment area in hectares

## MANAGEMENT COMMANDS

1. Process Unprocessed Datasets
   python manage.py process_lidar_datasets
   Finds and processes all unprocessed or failed datasets

2. Reprocess All Datasets
   python manage.py process_lidar_datasets --all
   Reprocesses all datasets, including completed ones

3. Process Specific Dataset
   python manage.py process_lidar_datasets --dataset-id=5
   Processes dataset with ID 5

## ERROR HANDLING

- Invalid file format: Returns 400 Bad Request
- Processing errors: Dataset status set to 'error'
- File not found: Returns FileNotFoundError
- Database errors: Returns 500 Internal Server Error

Check logs for details:
tail -f logs/django.log
