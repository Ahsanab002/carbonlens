QUICK START GUIDE - LIDAR BACKEND
==================================

## 1. VERIFY INSTALLATION

cd C:\biomass-vision-main\backend
.\venv\Scripts\Activate.ps1
python manage.py check
# Should output: "System check identified no issues (0 silenced)."

## 2. RUN TESTS

python manage.py test lidar_app -v 2
# Should output: "Ran 11 tests in X.XXXs - OK"

## 3. START DEVELOPMENT SERVER

python manage.py runserver
# Server runs on http://localhost:8000

## 4. TEST ENDPOINTS

### Option A: Using cURL (Windows PowerShell)

# List all datasets
Invoke-WebRequest -Uri "http://localhost:8000/api/lidar/datasets/" -Method GET

# List compartments
Invoke-WebRequest -Uri "http://localhost:8000/api/lidar/compartments/" -Method GET

# List biomass estimates
Invoke-WebRequest -Uri "http://localhost:8000/api/lidar/biomass/" -Method GET

### Option B: Using Postman/Thunder Client
1. Import these endpoints into Postman
2. Set Authorization header if needed (currently open)
3. Test each endpoint

### Option C: Using Backend API Directly

# Create a test LAS file upload script (Python):

import requests

files = {'file': open('test.las', 'rb')}
data = {
    'name': 'Test Dataset',
    'description': 'My first upload'
}

response = requests.post(
    'http://localhost:8000/api/lidar/datasets/upload/',
    files=files,
    data=data
)
print(response.json())

## 7. TROUBLESHOOTING

### Database Issues
python manage.py migrate
python manage.py makemigrations

### File Upload Issues
- Check MEDIA_ROOT exists: C:\biomass-vision-main\backend/media/lidar_uploads/
- Check file permissions
- Verify file is valid LAS/LAZ format

### Processing Errors
tail -f logs/django.log

### Clear Database (DEVELOPMENT ONLY)
python manage.py flush --no-input  # Deletes all data!

## 11. CORS CONFIGURATION

Already configured in settings.py:
- http://localhost:5173 (Vite frontend)
- http://127.0.0.1:5173

To add more origins:
1. Edit config/settings.py
2. Update CORS_ALLOWED_ORIGINS list
3. Restart server

## 12. LOGS

View application logs:

- Application logs stored in logs/django.log
- Database logs in PostgreSQL

## 13. PERFORMANCE TIPS

- Large files (>100MB) may take 30+ seconds
- Point cloud automatically downsampled to 100k points
- Use database indexes for faster queries
- Consider Celery for async processing of very large files
